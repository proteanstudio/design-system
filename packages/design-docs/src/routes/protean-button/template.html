Button

Buttons allow the user to perform actions in a given page or context, typically in forms, dialogs, or contextual menus.

Best practices

Use clear, concise button labels.

  • Button labels should indicate what will happen when the button is clicked.
  • Start with a verb (e.g. "Create", "Save", "Remove"), and supplement with a noun when you need additional clarity (e.g. "Create folder", "Save address", "Remove user").

Use only one primary button in a given page or view.

  • The primary button has the greatest visual prominence, so reserve it for the most important action on the page.
  • If there are several equally-important actions on a page, use the secondary style for all buttons on that page.

Use consistent button alignment.

  • Consistent button alignment reduces cognitive load and interaction cost.
  • Once you have the button alignment convention that's best for your design, stick to it.
  • Our alignment convention, for example, is to right-align buttons to their respective content, with priority actions to the right.

Use buttons for actions, and use links for navigation.

  • Buttons and links communicate distinct meaning both to users and to assistive technology.
  • Use a button to allow the user to interact with the current page or view.
  • Use a link to allow the user to go to a different page or view.

Take extra care with icon buttons

  • Icon buttons don't have visible labels, so users must rely on context and experience to understand the button's purpose.
  • Always include a descriptive a11yLabel for icon buttons to ensure they can be understood by users who rely on assistive technology.

API

Properties

In the following examples, element refers to an instance of protean-button (i.e. element = document.querySelector('protean-button#myButton')).

  • variant

    "primary" | "secondary | "icon"
    Attribute name: variant
    The visual style of the button. icon buttons should only be used in conjunction with protean-icon and should be used with an a11yLabel.
    element.variant = 'primary';

  • disabled

    boolean
    Attribute name: disabled
    Disables button interaction.
    element.disabled = true;

  • a11yLabel

    string
    Attribute name: a11y-label
    The label read by assistive technologies for the input; a11yLabel is used here instead of ariaLabel to prevent double readings of the text by those technologies. Note: This should not be set when the button has text rendered in it.
    element.a11yLabel = 'Field aria-label';

CSS variables

Listed below are the several CSS variables that can be adjusted to alter the protean-button element's appearance along with the default values set in the protean-styles package:

  • --protean-button-height

    The height of all buttons; typically bound to minimum touch zone size (42px).
    --protean-button-height: var(--touch-zone-size);

  • --protean-button-disabled-opacity

    The opacity of the button while it is disabled.
    --protean-button-disabled-opacity: var(--disabled-opacity);

  • --protean-button-border-radius

    The border radius of all button variants.
    --protean-button-border-radius: 21px;

  • --protean-button-primary-background-color

    The background color of the primary button.
    --protean-button-primary-background-color: var(--action-color);

  • --protean-button-primary-text-color

    The text color of the primary button.
    --protean-button-primary-text-color: var(--action-contrast-color);

  • --protean-button-primary-border-color

    The border color of the primary button.
    --protean-button-primary-border-color: transparent;

  • --protean-button-primary-hover-background-color

    The background color of the primary button while it is hovered or focused.
    --protean-button-primary-hover-background-color: var(--action-hover-color);

  • --protean-button-primary-hover-text-color

    The text color of the primary button while it is hovered or focused.
    --protean-button-primary-hover-text-color: var(--action-hover-contrast-color);

  • --protean-button-primary-hover-border-color

    The border color of the primary button while it is hovered or focused.
    --protean-button-primary-hover-border-color: transparent;

  • --protean-button-secondary-background-color

    The background color of the secondary button.
    --protean-button-secondary-background-color: transparent;

  • --protean-button-secondary-text-color

    The text color of the secondary button.
    --protean-button-secondary-text-color: inherit;

  • --protean-button-secondary-border-color

    The border color of the secondary button.
    --protean-button-secondary-border-color: var(--action-color);

  • --protean-button-secondary-hover-background-color

    The background color of the secondary button while it is hovered or focused.
    --protean-button-secondary-hover-background-color: var(--protean-button-primary-hover-background-color);

  • --protean-button-secondary-hover-text-color

    The text color of the secondary button while it is hovered or focused.
    --protean-button-secondary-hover-text-color: var(--protean-button-primary-hover-text-color);

  • --protean-button-secondary-hover-border-color

    The border color of the secondary button while it is hovered or focused.
    --protean-button-secondary-hover-border-color: var(--protean-button-primary-hover-border-color);

  • --protean-button-icon-size

    The width of icon buttons; typically matches the minimum touch zone size.
    --protean-button-icon-size: var(--touch-zone-size);

  • --protean-button-icon-background-color

    The background color of the icon button.
    --protean-button-icon-background-color: transparent;

  • --protean-button-icon-stroke-color

    The stroke color of the icon with the icon button.
    --protean-button-icon-stroke-color: currentColor;

  • --protean-button-icon-hover-background-color

    The background color of the icon button while hovered or focused.
    --protean-button-icon-hover-background-color: rgba(255, 95, 64, 0.2);

  • --protean-button-icon-hover-stroke-color

    The stroke color of the icon with the icon button while hovered or focused.
    --protean-button-icon-hover-stroke-color: currentColor;
