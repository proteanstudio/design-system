Best practices

Tell the user what happened, and what to do next.

  • If the user needs to take action in response to a message, give clear, specific instructions about what to do.

Use the right level of urgency.

  • Messages are interruptive; the level of interruption should match the urgency of the information.
  • Informational and success messages rarely require immediate attention, so level="status" is appropriate.
  • Warning and error messages often demand timely action, so use level="alert" to give it the necessary priority.

Use clear, polite language.

  • Be specific, be concise, and don't blame the user.

Use messages thoughtfully

  • Messages aren't effective when they are ignored; don't annoy or overwhelm the user with too many interruptions.

API

Properties

In the following examples, element refers to an instance of protean-message (i.e. element = document.querySelector('protean-message#myMessage')).

  • type

    "info" (default) | "success" | "warning" | "error"
    Attribute name: type
    The visual style of the message. Controls the icon and color used to communicate the intent of the message.
    element.type = 'success';

  • level

    "alert" (default) | "status"
    Attribute name: level
    Controls the role assigned to the message list, dictating the verbosity of assistive technologies.
    element.level = 'status';

CSS variables

Listed below are the CSS variables that can be adjusted to alter the appearance of protean-message. protean-message also inherits global status colors. The default values set in the protean-styles package are shown in the code snippets.

  • --protean-message-margin

    The margin around the protean-message element.
    --protean-message-margin: 1rem 0;

  • --protean-message-background-color

    The background color of the message container.
    --protean-message-background-color: #2a2a2a;

  • --protean-message-text-color

    The text color of the message container.
    --protean-message-text-color: #fcfcfc;

  • --protean-message-border-color

    The border color of the message container.
    --protean-message-border-color: #353535;

  • --protean-status-info

    The color used for informational statuses.
    --protean-status-info: #0965b7;

  • --protean-status-info-contrast

    The color used as a contrasting foreground or background for informational statuses.
    --protean-status-info-contrast: #ffffff;

  • --protean-status-success

    The color used for success statuses.
    --protean-status-success: #1d820f;

  • --protean-status-success-contrast

    The color used as a contrasting foreground or background for success statuses.
    --protean-status-success-contrast: #ffffff;

  • --protean-status-warning

    The color used for warning statuses.
    --protean-status-warning: #e6c400;

  • --protean-status-warning-contrast

    The color used as a contrasting foreground or background for warning statuses.
    --protean-status-warning-contrast: #1a1a1a;

  • --protean-status-error

    The color used for error statuses.
    --protean-status-error: #ab0000;

  • --protean-status-error-contrast

    The color used as a contrasting foreground or background for error statuses.
    --protean-status-error-contrast: #ffffff;
