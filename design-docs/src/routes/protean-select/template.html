Select

Selects present a collapsible list of options from which the user can make a single selection (the default), or multiple selections (when using the multi-select variant).

Best practices

Selects, radio buttons, and checkboxes all provide the same fundamental function: choosing from a list of options. Consider these questions before using a select in your design:

How many options are in the list?

  • If there are five or fewer options, radio buttons (for a single selection) or checkboxes (for multiple selections) are less demanding on the userâ€™s memory and outperform selects in usability tests.

Are users likely to know what option they want before seeing the list?

  • If users are familiar with the list of options and how they are ordered, a select may be appropriate. For highly familiar option sets, like US states, it may be easier simply to type the answer.
  • If users likely need to browse the options before choosing, radio buttons or checkboxes support browsing better than selects.

Is screen space constrained?

  • When screen space is limited, a select can be an efficient way to allow the user to choose from a list of options.

API

Select Properties

In the following examples, element refers to an instance of protean-select (i.e. element = document.querySelector('protean-select#mySelect')).

  • value

    string
    The value of the select when not using a multiple select. Should match the value of one of the nested options.
    element.value = 'option1';

  • selectedOptions

    string[]
    The list of selected options when using a multiple select. Individual items should match the values of one of the nested options.
    element.selectedOptions = ['option1', 'option2'];

  • multiple

    boolean
    Attribute name: multiple
    Allows selection of multiple options
    element.multiple = true;

  • label

    string
    Attribute name: label
    The visible label for the select.
    element.label = 'Field label';

  • a11yLabel

    string
    Attribute name: a11y-label
    The label read by assistive technologies for the select; a11yLabel is used here instead of ariaLabel to prevent double readings of the text by those technologies. Note: when label is set, this value will not be bound to the field.
    element.a11yLabel = 'Field aria-label';

  • optional

    boolean
    Attribute name: optional
    Marks the field as optional to the user by appending "(optional)" to the visible label and to assistive technologies by setting aria-required="false".
    element.optional = true;

  • errors

    string[]
    A list of errors used to show the field in an error state. The contents of the errors are not visible to the user.
    element.errors = [ 'Error #1', ' Error #2' ];

  • disabled

    boolean
    Attribute name: disabled
    Disables the field and prevents user interaction.
    element.disabled = true;

Optgroup Properties

  • label

    string
    Attribute name: label
    The visible label for the group.
    element.label = 'Group label';

  • disabled

    boolean
    Attribute name: disabled
    Disables the group and prevents any options with it from being selected.
    element.disabled = true;

Option Properties

  • value

    string
    Attribute name: value
    The value of the option. This value is emitted when the option is selected.
    element.value = 'option1';

  • label

    string
    Attribute name: label
    The text shown in the select when this option is selected. If not populated, the select will fall back to the option's inner text and further to its value.
    element.label = 'Group label';

  • disabled

    boolean
    Attribute name: disabled
    Disables the options and prevents it from being selected.
    element.disabled = true;

  • selected

    boolean
    Attribute name: selected
    Indicates whether an option is selected. Can be used to pre-select options on render but once value or selectedOptions is set on the select, the properties of the select override any user-defined properties on the option.

CSS variables

Listed below are the CSS variables that can be adjusted to alter the appearance of protean-select, protean-optgroup, and protean-option. protean-select inherits styles from protean-input and dropdown styles that will also be used in future components. The default values set in the protean-styles package are shown in the code snippets.

  • --protean-select-margin

    The margin around the protean-select element.
    --protean-select-margin: 1rem;

  • --protean-select-separator-color

    The color of the line between the select field and its dropdown.
    --protean-select-separator-color: #535353;

  • --protean-optgroup-separator-color

    The color of the line used to distinguish the optgroup.
    --protean-optgroup-separator-color: var(--protean-select-separator-color);

  • --protean-dropdown-background-color

    The background color of the select dropdown.
    --protean-dropdown-background-color: var(--body-background-color);

  • --protean-dropdown-text-color

    The text color of content within the select dropdown.
    --protean-dropdown-text-color: var(--body-text-color);

  • --protean-dropdown-border-color

    The border color of the select dropdown.
    --protean-dropdown-border-color: currentColor;

  • --protean-dropdown-border-width

    The border width of the select dropdown.
    --protean-dropdown-border-width: 0 1px 1px;

  • --protean-dropdown-box-shadow

    The box shadow style of the select dropdown.
    --protean-dropdown-box-shadow: 0 3px 5px -2px rgba(255, 255, 255, 0.7);

  • --protean-dropdown-hover-background-color

    The background color of an option when hovered or active.
    --protean-dropdown-hover-background-color: #353535;

  • --protean-dropdown-hover-text-color

    The text color of an option when hovered or active.
    --protean-dropdown-hover-text-color: #353535;
